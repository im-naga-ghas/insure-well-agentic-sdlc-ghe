# Azure DevOps Integration Guide

This guide covers connecting Azure DevOps to GitHub, setting up the Azure DevOps MCP server in VS Code, and working with work items via Copilot agents.

---

## Part 1 — Create a Personal Access Token (PAT) in Azure DevOps

A PAT is required for API access and MCP server authentication.

1. Sign in to [dev.azure.com](https://dev.azure.com) and select your organization.
2. Click your profile icon (top-right) → **Personal access tokens**.
3. Click **+ New Token**.
4. Fill in the details:
   - **Name:** `copilot-mcp` (or any descriptive name)
   - **Organization:** select your org
   - **Expiration:** set an appropriate duration (e.g., 90 days)
   - **Scopes:** select **Custom defined**, then enable:
     - `Work Items` — Read & Write
     - `Code` — Read
     - `Build` — Read
     - `Project and Team` — Read
5. Click **Create** and **copy the token immediately** — it will not be shown again.
6. Store it securely (e.g., in your password manager or as an environment variable):

```bash
export AZURE_DEVOPS_PAT="<your-token-here>"
```

## Part 2 — Create a Service Connection in Azure DevOps

A service connection lets Azure DevOps pipelines authenticate with external services (e.g., GitHub, Docker Hub, Azure).

1. In your Azure DevOps project, go to **Project Settings** (bottom-left gear icon).
2. Under **Pipelines**, select **Service connections**.
3. Click **+ New service connection**.
4. Choose the connection type (e.g., **GitHub**, **Docker Registry**, **Azure Resource Manager**).
5. For a **GitHub** connection:
   - Select **GitHub (personal access token)** or **OAuth**.
   - For PAT: paste a GitHub PAT with `repo` and `read:org` scopes.
   - **Connection name:** `github-insurewell`
   - Check **Grant access permission to all pipelines**.
6. Click **Save** to create the connection.

## Part 3 — Establish a GitHub ↔ Azure DevOps Connection

### Link GitHub Repository to Azure DevOps Project

1. In your Azure DevOps project, navigate to **Project Settings** → **GitHub connections** (under Boards or Pipelines).
2. Click **Connect your GitHub account** (OAuth) or **Add connection** (PAT).
3. Authorize the Azure Boards GitHub App or enter a GitHub PAT.
4. Select the GitHub repositories to connect (e.g., `im-naga-ghas/insure-well-agentic-sdlc-ghe`).
5. Click **Save**.

Once linked, GitHub commits and PRs can be associated with Azure Boards work items by including the work item ID in commit messages:

```
git commit -m "feat: add user auth #AB-123"
```

### Link Azure Pipelines to GitHub Repository

1. Go to **Pipelines** → **New pipeline**.
2. Select **GitHub** as the source.
3. Choose the repository (`insure-well-agentic-sdlc-ghe`).
4. Authorize using the service connection created in Part 2.
5. Configure the pipeline using the existing `.github/workflows/` as a reference or create a new `azure-pipelines.yml`.

## Part 4 — Add the Azure DevOps MCP Server & GitHub MCP Server in VS Code

The Azure DevOps MCP server exposes work items, pipelines, repos, and wiki APIs directly to Copilot agents.

### Step 1 — Install Prerequisites

```bash
# Verify Node.js is installed (v18+ recommended)
node --version

# Install the Azure DevOps MCP package globally (if using stdio mode)
# Install the GitHub MCP package globally (if using stdio mode)
```

Use these official repositories for setup and reference:
- [Azure DevOps MCP server](https://github.com/mcp/microsoft/azure-devops-mcp)
- [GitHub MCP server](https://github.com/mcp/github/github-mcp-server)

### Step 2 — Configure the MCP Server in VS Code

Once installed clikc on Gear icon in VS Code -> 
Click on Start Server
![alt text](images/Start%20MCP.png)
### Step 3 — Enable the MCP Server in VS Code

1. Open the Command Palette (`Cmd+Shift+P`).
2. Run **"MCP: List Servers"** — `azure-devops` should appear.
3. Click **Start** next to `azure-devops` (or it starts automatically on first use).
4. Confirm the status indicator shows green / running.

### Step 4 — Verify the Connection

In Copilot Chat (Agent mode), run a quick test:

```
List all projects in my Azure DevOps organization.
```

Or check work items in a specific project:

```
Show me the open work items in project <your-project-name>.
```

## Part 5 — Create & Pull Work Items via Copilot

### Create a Work Item

In Copilot Chat (Agent mode), describe the work item and let the Azure DevOps MCP tools handle it:

```
Create a new User Story in Azure DevOps project "InsureWell" with:
- Title: "As a policyholder, I want to log in so my policies are private"
- Description: "Implement username/password authentication for the React frontend and Spring Boot backend"
- Priority: 1
- Tags: auth, backend
```

Or create a Bug:

```
Create a Bug in project "InsureWell":
- Title: "Claim form does not validate negative amounts"
- Repro steps: "Enter -100 in the amount field and submit"
- Severity: 2
```

### Pull / Query Work Items

Retrieve open work items assigned to you:

```
Show me all active work items assigned to me in project "InsureWell".
```

Query by iteration or sprint:

```
List all User Stories in the current sprint for project "InsureWell".
```

Filter by state or tag:

```
Get all work items in "InsureWell" with state "Active" and tag "auth".
```

### Update a Work Item

```
Update work item #42 in "InsureWell":
- Set state to "Resolved"
- Add comment: "Implemented in PR #17"
```

### Link a Work Item to a GitHub PR or Commit

From within Copilot Chat:

```
Link work item #42 in Azure DevOps project "InsureWell" to 
GitHub PR #17 in im-naga-ghas/insure-well-agentic-sdlc-ghe.
```

Or include the work item ID in commit messages to auto-link (requires Part 3 connection):

```bash
git commit -m "fix: validate claim amount field AB#42"
```

| Task | Command / Location |
|---|---|
| Create PAT | dev.azure.com → Profile → Personal access tokens |
