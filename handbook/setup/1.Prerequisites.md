# Prerequisites

> **New to this workshop?** Complete every item in this checklist before the session starts. Each step is explained below. If you get stuck, ask a facilitator.

---

## Required Accounts & Access

- [ ] **GitHub Enterprise** account with access to the workshop organization
- [ ] **GitHub Copilot** subscription (active — confirm with your admin if unsure)
- [ ] **VS Code** 1.90+ installed on your laptop

> **Note:** Azure DevOps access is optional and only needed for the ADO integration segment. If you are unsure, skip it for now — the core workshop runs entirely on GitHub.

---

## Step 1 — Install Local Tools

Install each tool and verify the version with the commands below.

| Tool | Minimum version | Download |
|---|---|---|
| Java (JDK) | 17 | https://adoptium.net |
| Maven | 3.9 | https://maven.apache.org/download.cgi |
| Node.js | 18 | https://nodejs.org |
| Git | Any recent | https://git-scm.com |

```bash
# Run these in your terminal to confirm versions
java --version    # should print 17.x or higher
mvn --version     # should print 3.9.x or higher
node --version    # should print v18.x or higher
npm --version     # should print 9.x or higher
git --version     # should print any recent version
```

---

## Step 2 — Configure Git Identity

Git needs to know who you are before you can make commits.

```bash
git config --global user.name  "Your Name"
git config --global user.email "you@example.com"
```

---

## Step 3 — Install VS Code Extensions

Open VS Code and install the following extensions from the **Extensions Marketplace** (`Cmd+Shift+X` / `Ctrl+Shift+X`). Search by name:

**Required:**
- `GitHub Copilot`
- `GitHub Copilot Chat`
- `GitHub Pull Requests and Issues`
- `Extension Pack for Java`
- `Spring Boot Extension Pack`
- `ES7+ React/Redux/React-Native snippets`

> Tip: After installing all extensions, restart VS Code once to make sure they activate correctly.

---

## Step 4 — Create a GitHub PAT

A Personal Access Token (PAT) lets the MCP server authenticate with GitHub on your behalf.

1. Go to **GitHub** → your profile icon (top-right) → **Settings**.
2. Scroll to **Developer settings** → **Personal access tokens** → **Fine-grained tokens** (or **Tokens (classic)**).
3. Click **Generate new token**.
4. Set a descriptive name (e.g., `workshop-mcp`) and an expiry of at least 7 days.
5. Enable these scopes: `repo`, `read:org`, `workflow`.
6. Click **Generate token** and **copy it immediately** — it is shown only once.
7. Store it safely (e.g., in a notes app or password manager) until you need it during the MCP setup step.

---

## Step 5 — Verify GitHub Access

```bash
# Replace YOUR_PAT with the token you just created
curl -H "Authorization: token YOUR_PAT" https://api.github.com/user
# Expected: JSON with your GitHub username
```
