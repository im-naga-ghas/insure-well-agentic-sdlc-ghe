# Playwright MCP Workshop Guide — InsureWell

> Read this guide before using the Playwright MCP server with GitHub Copilot in this repository.

This workshop is organized for hands-on delivery against the **InsureWell** health insurance management app (React + Spring Boot). It starts with setup, moves into quick validation, then walks through guided MCP exercises covering the Dashboard and Claims UI.

---

## 1. Workshop Goal

By the end of this lab, participants should be able to:

- understand what Playwright MCP adds on top of normal Playwright test generation
- configure the Playwright MCP server in VS Code
- start the InsureWell full-stack application and verify the configured base URL
- use MCP browser tools to navigate, click, fill forms, inspect, and validate InsureWell UI behavior
- connect interactive MCP validation with the repo's Playwright test suite in `src/frontend/tests/`

---

## 2. What Is Playwright MCP?

The Playwright MCP (Model Context Protocol) server exposes browser automation
capabilities to AI agents as callable tools. When configured in VS Code, Copilot
can navigate pages, click elements, fill forms, take screenshots, and evaluate
JavaScript — all within the agent conversation.

This enables two modes of operation:

| Mode | What it does |
| --- | --- |
| Generate | Agent writes Playwright spec files. |
| Generate + Execute | Agent writes tests and can also validate behavior interactively against the running app. |

## 3. Application Architecture

| Layer | Technology | URL |

| Frontend | React 18 (Create React App) | `http://localhost:3000` |

## 5. Install And Configure Playwright MCP

### Option A: Global install

```bash
npm install -g @playwright/mcp
```

Verify installation:

```bash
npx @playwright/mcp --version
```

### Option B: Workspace-driven execution (already configured)

This repo uses `npx @playwright/mcp@latest` inside `.vscode/mcp.json`, so a global install is optional.

### Verify VS Code picks it up

1. Open Copilot Chat.
2. Click the **Tools** (plug) icon in the chat input bar.
3. Confirm the Playwright `browser_*` tools are listed.
4. If not visible, reload the VS Code window (`Cmd+Shift+P` → `Developer: Reload Window`) and check the Output panel for errors.

## 6. Repository Settings

### MCP config — `.vscode/mcp.json`

```json
{
  "servers": {
    "microsoft/playwright-mcp": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "@playwright/mcp@latest"
      ],
      "env": {
        "PLAYWRIGHT_BASE_URL": "http://localhost:3000"
      }
    }
  },
  "inputs": []
}
```

### Playwright config — `src/frontend/playwright.config.ts`

```ts
use: {
  baseURL: 'http://localhost:3000',
  trace: 'on-first-retry',
  screenshot: 'only-on-failure'
}
```

If the app host or port changes, update both files.

## 7. Available Tool Names

When the MCP server is active, these browser tools are available.

### Navigation

- `browser_navigate` — Navigate to a URL
- `browser_navigate_back` — Go back in browser history
- `browser_resize` — Resize the browser window
- `browser_close` — Close the current page

### Interaction

- `browser_click` — Click a web page element
- `browser_fill_form` — Fill multiple form fields at once
- `browser_type` — Type text into an editable element
- `browser_select_option` — Select a dropdown option
- `browser_handle_dialog` — Handle browser alert/confirm/prompt
- `browser_hover` — Hover over an element
- `browser_press_key` — Press a keyboard key
- `browser_file_upload` — Upload one or more files

### Inspection

- `browser_snapshot` — Accessibility-style snapshot of the current page
- `browser_take_screenshot` — Screenshot of the current page
- `browser_evaluate` — Evaluate JavaScript on the page
- `browser_console_messages` — Return browser console output
- `browser_network_requests` — Return network requests since page load

### Context

- `browser_tabs` — List, create, close, or select a browser tab

## 8. Selector Convention For This App

Always use `data-testid` attributes. Do not use CSS class or positional selectors.

**Correct:**

```text
[data-testid='add-policy-btn']
data-testid=new-claim-btn
```

**Wrong:**

```text
.btn-primary
button:nth-child(2)
```

## 12. Guided Hands-On Exercises

Use these prompts directly in Copilot Chat with Playwright MCP enabled.

### Stage A: Navigation And Visibility

#### 1. Inspect the Dashboard

```text
Navigate to http://localhost:3000 and give me an accessibility snapshot of the page.
```

Tools: `browser_navigate`, `browser_snapshot`

Expected: navbar with Dashboard and Claims links, policy tabs, stat cards.

#### 2. Switch to Claims via navigation

```text
Navigate to http://localhost:3000, click [data-testid='nav-claims'], and confirm the Claims page heading is visible.
```

Tools: `browser_navigate`, `browser_click`, `browser_snapshot`

---

### Stage B: Policy Modal

#### 3. Open the Add Policy modal

```text
Navigate to http://localhost:3000, click [data-testid='add-policy-btn'], and confirm the policy modal is visible.
```

Tools: `browser_click`, `browser_snapshot`

#### 4. Close the modal with Cancel

```text
Navigate to http://localhost:3000, open the Add Policy modal, then click [data-testid='cancel-policy-btn'] and verify the modal closes.
```

Tools: `browser_click`, `browser_snapshot`

#### 5. Trigger validation error in the policy form

```text
Navigate to http://localhost:3000, open the Add Policy modal, click [data-testid='save-policy-btn'] without filling any fields, and tell me what error appears.
```

Tools: `browser_click`, `browser_snapshot`

---

### Stage C: Add A Policy

#### 6. Fill and submit the Add Policy form

```text
Navigate to http://localhost:3000 and add a new policy with these values:
Holder Name: Jane Doe
Plan Name: Gold Plan
Coverage Amount: 75000
Status: active
Start Date: 2026-01-01
End Date: 2026-12-31
Click Save and confirm the modal closes and the new policy tab appears.
```

Tools: `browser_click`, `browser_fill_form`, `browser_snapshot`

#### 7. Open the New Claim form

## 13. Running The Playwright Test Suite

Ensure the app is running on port 3000 first, then from `src/frontend/`:

```bash
cd src/frontend
npm run test:e2e                  # run all specs headlessly
npm run test:e2e:headed           # run with browser visible
npm run test:e2e:dashboard        # dashboard tests only
npm run test:e2e:claims           # claims tests only
npm run test:e2e:report           # open HTML report
npx playwright test --reporter=list
```

Test files:

- `src/frontend/tests/dashboard.spec.ts` — navigation, policy modal, form validation, add policy
- `src/frontend/tests/claims.spec.ts` — new claim, filter, status update, delete, screenshot

## 14. Troubleshooting

### MCP server does not appear in Copilot

1. Check `.vscode/mcp.json` syntax (valid JSON, correct indentation).
2. Reload VS Code window (`Cmd+Shift+P` → `Developer: Reload Window`).
3. Open the Output panel and select the MCP or Copilot channel for errors.
4. Run `npx @playwright/mcp --version` to confirm the package resolves.

### Backend API errors in the UI

- Confirm Spring Boot is running: `curl http://localhost:8080/api/policies`
- Check terminal for Java startup errors (port conflict, missing Java version).
- The frontend shows `"Failed to load data from backend"` if the API is unreachable.

### Navigation fails immediately in MCP

- Confirm the React app is running: `curl http://localhost:3000`
- Confirm the MCP `PLAYWRIGHT_BASE_URL` matches `playwright.config.ts` `baseURL`.

### An element is not found

- Use `browser_snapshot` first to inspect current page state.
- Confirm the correct page is loaded (Dashboard vs. Claims).
- Use the exact `data-testid` from the selector table in Section 8.

### CLI tests fail but MCP interaction works

- Run `npm run test:e2e:headed` to watch the browser during test execution.
- Check for timing issues — the backend seed data may not be loaded yet.
